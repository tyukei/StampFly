# StampFly 公式ファーム 書き込み手順・まとめ

このドキュメントは、StampFly を「出荷時と同じ状態（M5Stack 公式ファーム）」に戻すための手順と、
そこに至るまでに判明した重要事項をまとめたもの。

---

## 0. 結論（まず読む）

- **出荷時ファーム = 公式リポジトリ [`m5stack/M5StampFly`](https://github.com/m5stack/M5StampFly)**
- **純正 ATOMJoy（リモコン）とペアリングできるのは公式ファームだけ**
- `M5Fly-kanazawa/StampFly`（金沢の教育向けフォーク）は飛行制御ロジックの参照には使えるが、
  **純正 ATOMJoy とは ESP-NOW のプロトコル/チャンネルが食い違い、ペアリングできない**
  （ATOMJoy の液晶表示が変わらず、飛べない）。
- ローカルの `StampFly` リポジトリ（この repo）は dashboard / GUI / EEG→MQTT 機能で派生済みなので、
  出荷時の飛行制御は残っていない。復元は必ず外部の公式リポジトリから行う。

---

## 1. 公式ファームの書き込み手順（メイン）

### 前提
- macOS + PlatformIO（`pio`）がインストール済み
- StampFly の電源スイッチ ON、データ通信対応 USB ケーブルで Mac に接続

### 手順

```bash
# 1. 公式リポジトリを clone
git clone https://github.com/m5stack/M5StampFly.git
cd M5StampFly

# 2. 接続ポートを確認（ESP32-S3 ネイティブ USB）
ls /dev/cu.usbmodem*
#   → /dev/cu.usbmodem101 のように出る（番号は接続ごとに変わりうる。過去は 1101）

# 3. ビルド＆書き込み（bootloader / partition / app をすべて書き込む）
pio run --target upload --upload-port /dev/cu.usbmodem101
```

- **書き込み成功の判定は PlatformIO の exit code 0**。
- 公式リポには `firmware/*.bin`（ビルド済みバイナリ）も同梱されているが、
  ソースから `pio run -t upload` した方が bootloader/partition まで含めて完全に書き込まれる。

### 起動確認（任意）
シリアルモニターで起動ログを見て、以下が出れば正常：
```
... (センサー初期化ログ) ...
ESPNow Init Success2
ESP-NOW Ready.
Finish StampFly init!
Enjoy Flight!
```
```bash
pio device monitor -p /dev/cu.usbmodem101 -b 115200
```

## 2. トラブルシューティング

### USB ポート（`/dev/cu.usbmodem*`）が出てこない
1. StampFly の**電源スイッチが ON** か確認
2. **データ通信対応 USB ケーブル**を使う（充電専用ケーブルは NG）。別のケーブルに交換して試す
3. USB ハブ経由をやめて本体直挿しにする
4. それでも出なければ**ダウンロードモード**へ：
   `BOOT` ボタンを押しながら `RESET` を押して離す → `BOOT` を離す → USB 挿し直し

### auto mode で書き込みがブロックされる
- 外部リポジトリの clone → ビルド → 書き込みは、分類器に一度ブロックされることがある（ソース確認のため）。
  復元元（`m5stack/M5StampFly`）を確認すれば続行できる。

---

## 3. ペアリング手順（純正 ATOMJoy）

公式ファーム書き込み後、初めて飛ばすときは ATOMJoy とペアリングが必要。

1. **ATOMJoy**：液晶パネルを押しながら電源 ON（ペアリング待ち受けモードに入る）
2. **その待ち受け中に** StampFly を起動 / リセット
   （StampFly は起動〜数秒後に自分の MAC を約 1 秒間ブロードキャストする）
3. **ATOMJoy の液晶表示が変われば完了** 🎉

### ポイント（つまずきやすい所）
- **順番とタイミングが肝**。StampFly の MAC ブロードキャストは「起動直後の一瞬」だけ。
  すでに虹色イルミネーション（＝起動完了・待機状態）になった後では電波は飛んでいないので拾えない。
- 正しくは「ATOMJoy を待ち受けにしてから StampFly を起動（リセット）」する。
- USB 経由なら Python + pyserial で DTR/RTS リセットをかけ、ブロードキャストの瞬間を検知して
  タイミングを合わせられる（`ESPNow Init Success`〜`ESPNow Init Success2` の間が送信中）。

## 4. LED ステータスの意味

| 状態 | LED |
|---|---|
| 起動直後 | 白 |
| キャリブレーション中（**触らない**） | 紫 |
| 飛行待機 OK | 虹色イルミネーション |
| 飛行・Stabilize モード | 黄 |
| 飛行・Acro モード | ピンク系 |
| 高度維持モード | 紫系 |
| リモコン切断エラー（飛行中） | 赤 |
| 低バッテリー | 水色点滅 |

飛ばす流れ：水平に置く（白）→ 紫（キャリブレーション、触らない）→ 虹色（待機）
→ 左スティック押し込みで Arming（黄）→ 左スティック上でスロットル上昇。

## 5. 搭載センサーと取得できる情報

制御ループは **400Hz（2.5ms 周期）**。

### 使用中
- **BMI270（IMU）**：3軸加速度 `Accel_x/y/z`[g]、3軸角速度 `Roll/Pitch/Yaw_rate`[rad/s]、
  推定姿勢角 `Roll/Pitch/Yaw_angle`、クォータニオン `Quat`、衝撃検知 `Over_g/OverG_flag`、鉛直加速度 `Az`
- **VL53LX ToF ×2**：下向き `Range`[mm]（高度）、**前向き** `RangeFront`[mm]（前方障害物）
- **高度カルマン（alt_kalman）**：`Altitude`[m]、`Alt_velocity`[m/s]
- **INA3221**：`Voltage`（バッテリー電圧、低電圧検知）

### 載っているが未使用（＝追加ハード不要ですぐ着手できる発展テーマ）
- **BMP280（気圧センサー）**：気圧・温度 → 高高度での高度推定
- **BMM150（地磁気センサー）**：方位（コンパス）→ ヘディングホールド、ヨードリフト補正
- **INA3221 の電流測定**：消費電力・バッテリー残量%・飛行効率解析

### 非搭載
- **カメラ・光学フロー（PMW3901）は無い** → 映像処理・水平位置ホールドは単体では不可。
  位置制御をやりたいなら **PMW3901（光学フロー）追加が本命**（カメラより軽く確実）。

## 6. 参考：発展の方向性（軸）

- **軸A 自律制御を深める**：高度維持モード完成、前方 ToF で障害物回避/壁沿い、ヘディングホールド、自動離着陸
- **軸B 外部連携を尖らせる**：EEG→ドローン制御の本格化、Web ダッシュボードで飛行中 PID チューニング、ESP-NOW 編隊
- **軸C データ/シミュ基盤**：高頻度ログ → システム同定 → SIL シミュレータ → 強化学習
- **軸D ハード拡張**：PMW3901 光学フロー追加で屋内ホバリング、外付けカメラ/UWB

おすすめ：軸Aで「安全な自律ホバリングの土台」を作り、その上に軸Bのブレイン制御を載せる。
