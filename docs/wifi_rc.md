# スマホ(Android/iPhone)からのWi-Fiリモコン操作

純正ATOMコントローラ（ESP-NOW）の代わりに、**スマホのブラウザ**からStampFlyを操作するためのファームウェアです。アプリのインストールもインターネットも不要で、ドローン自身がWi-Fiアクセスポイントになります。

- 通信: ESP32 soft-AP + WebSocket（双方向・低遅延・ブローカ不要）
- できること: 仮想スティックによる飛行操作 / 各モーターの回転数(duty)個別制御 / センサー値のリアルタイム表示
- 対象ファーム: `src/wifi_rc_drone.cpp`（操作ページは `src/wifi_rc_page.h` に埋め込み）

> ⚠️ **安全**: 初回テストは必ずプロペラを外して行ってください。モーターは「ARM済み」かつ「通信が生きている」ときだけ回り、通信断・ブラウザを閉じる・STOPボタン・連続20秒で自動停止します。

## 書き込み手順

```bash
# ビルド
pio run -e wifi_rc

# StampFlyをUSB接続して書き込み
pio run -e wifi_rc -t upload

# シリアルモニタ（起動ログ・接続状況の確認）
pio device monitor -e wifi_rc
```

## 接続・操作手順

1. StampFlyに給電すると、Wi-Fiアクセスポイントが立ち上がります。
   - SSID: **`StampFly-RC`**
   - パスワード: **`stampfly123`**
2. スマホのWi-Fi設定でこのSSIDに接続します（この間スマホはインターネットに繋がりません）。
3. ブラウザ（Chrome/Safari）で **`http://192.168.4.1/`** を開きます。
   - ホーム画面に追加するとアプリのように起動できます（PWA）。
4. 画面上部にセンサー値（加速度3軸・角速度3軸・ロール/ピッチ姿勢角）が表示されます。
5. 操作モードを選びます。
   - **飛行モード**: 左スティック=スロットル(上下)/ラダー・ヨー(左右)、右スティック=エレベーター(上下)/エルロン・ロール(左右)
   - **モーターテスト**: FL/FR/RL/RR の4本のスライダーで各モーターの回転数(duty)を直接指定
6. **ARM** ボタンで発進準備、**STOP** で即停止（ディスアーム）。

## 飛行モードの操作（Mode 2 レイアウト）

画面下部の2つの仮想スティックで操作します。

```
┌─────────────┐  ┌─────────────┐
│   スロットル  │  │  エレベーター │   ← 上下
│  ┌───────┐  │  │  ┌───────┐  │
│  │  ●    │  │  │  │  ●    │  │
│  └───────┘  │  │  └───────┘  │
│ ラダー/ヨー  │  │ エルロン/ロール│   ← 左右
└─────────────┘  └─────────────┘
   左スティック        右スティック
```

### 左スティック

| 方向 | 操作 | 挙動 |
|------|------|------|
| 上下 | スロットル（上昇/下降） | **指を離した位置で保持**（ラッチ式）。下=停止、上=最大 |
| 左右 | ラダー / ヨー（機首を左右に回す） | 離すと中央に戻る |

### 右スティック

| 方向 | 操作 | 挙動 |
|------|------|------|
| 上下 | エレベーター / ピッチ（前後の傾き） | 離すと中央に戻る |
| 左右 | エルロン / ロール（左右の傾き） | 離すと中央に戻る |

> 左スティックの上下（スロットル）だけは離しても戻らず、指を離した高さを保持します。他の3軸は指を離すと自動でニュートラルに戻ります。

### 操作の流れ

1. **プロペラを外した状態で** **ARM** を押す（ボタンが赤く点滅＝始動）。
2. 左スティックをゆっくり上げてスロットルを上げる → 4基が回り始める。
3. 右スティックで傾き、左スティック左右で旋回。
4. 下げたいときは左スティックを下へ、止めるときは **STOP**（即ディスアーム）。

### 出力の内訳

- スロットル最大でも各モーター duty は `THROTTLE_SCALE`(0.60) まで。ロール/ピッチ混合は最大 `±MIX_GAIN`(0.15)、ヨーは `±YAW_GAIN`(0.12)。
- 全モーター最終的に `MAX_DUTY`(0.35) でクランプ。

### 飛ばす前のミキサー符号チェック（プロペラを外して）

ARMして各操作をしたとき、下表どおりにモーターが増速するか確認してください。逆であれば `src/wifi_rc_drone.cpp` の `drive_mix()` の符号を修正します。

| 操作 | 増速するモーター |
|------|------------------|
| ロール右（右スティック右） | 左側 FL・RL |
| ピッチ前（右スティック上） | 後側 RL・RR |
| ヨー右（左スティック右） | FR・RL |

センサー表示のロール/ピッチ角が、機体を手で傾けた向きと一致しているかも合わせて確認しておくと安心です。

## 安全パラメータ（`wifi_rc_drone.cpp` 冒頭で調整可能）

| 定数 | 既定値 | 意味 |
|------|--------|------|
| `MAX_DUTY` | 0.35 | モーターduty上限（各モーター） |
| `THROTTLE_SCALE` | 0.60 | スロットル1.0時の最大duty |
| `MIX_GAIN` / `YAW_GAIN` | 0.15 / 0.12 | 姿勢混合の寄与量 |
| `CMD_TIMEOUT_MS` | 600 | 無通信で自動停止するまでの時間 |
| `RUN_LIMIT_MS` | 20000 | ARM連続稼働の上限（超えると再ARMが必要） |

## 実装メモ

- モーターPWMは `flight_control.cpp` と同一（150kHz / 8bit、ピン FL=5 / FR=42 / RL=10 / RR=41、duty=255×d）。
- センサーはIMU(BMI270)を `imu.cpp` 経由で実読み。姿勢角は加速度＋ジャイロの相補フィルタで算出。
- 飛行モードのミキサー符号（roll/pitch/yaw）は機体により反転が必要な場合があります。**必ずプロペラを外して**各スティック操作時のモーター増減方向を確認し、必要なら `drive_mix()` の符号を調整してください。
- この環境は既存の飛行スタック（400Hz PID）とは独立した自己完結ビルドです。ATOMコントローラでの飛行に戻すにはデフォルト環境を書き込み直してください。
